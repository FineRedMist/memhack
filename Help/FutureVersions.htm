Future Versions

There are a couple of other features I would like to implement for MemHack. Some I've experimented with already and haven't solved yet:

  • Showing the account each process is running as
  • Supporting locking the value for a given address (so if it changes it will be reset back to the value specified)
  • Possibly a more general memory viewer so you can look at a region of memory around an address in a target application
  • Showing the user the list of memory regions
  • Fix a really rare race condition on exit causing an object is being disposed of twice
  • Fix an unusual performance issue for large numbers of addresses where a Find Next is dramatically less performant than a Find First.
